## Mentor Acceptance Criteria

### Project

- **A1** The HTML file is named `index.html`.
- **A2** The root of the project contains an `images` folder with all images.
- **A3** File names do not contain uppercase letters, Cyrillic characters, spaces, or transliteration. Names include only English letters and words.
- **A4** The source code is formatted using Prettier.
- **A5** All images and text content are taken from the provided layout.
- **A6** All raster images are optimized using Squoosh.
- **A7** The code is written following the project guidelines.

### Markup

- **B1** The markup is written in the `index.html` file.
- **B2** HTML markup is implemented for all layout elements.
- **B3** Tags are used according to their semantic meaning.
- **B4** HTML passes validation without errors.
- **B5** The markup includes tags defining the main page structure: `<header>`, `<main>`, and `<footer>`.
- **B6** The logo in the header and footer is a text link, not an image.
- **B7** The `<nav>` tag is used only once on the page — in the header.
- **B8** Header contacts (email and phone) are placed outside the `<nav>` tag.
- **B9** The `<h1>` tag is used only once on the page.
- **B10** The element with the text `Order Service` is a button with `type="button"`.
- **B11** Section titles are marked up using the `<h2>` tag.
- **B12** `<img>` tags include size attributes, at least `width`.
- **B13** `<img>` tags include an `alt` attribute with a short description of the image in the same language as the rest of the content.
- **B14** Images are exported from the layout in `jpg` format.
- **B15** Groups of similar elements are organized into `<ul>` lists.
